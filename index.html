Abstract:

Cloud storage outsourcing has become a popular application for enterprises and organizations to reduce the burden of maintaining big data in recent years. However, in reality, tenants may not entirely trust the cloud storage servers and may prefer to encrypt their data before uploading them to the cloud server in order to protect the data privacy. But how ever providing data privacy, as well as it needs to enhance access privilege also. In past years if user A encrypts the data and storing into cloud then later user B can decrypt the data from cloud with respective key directly. In this case if any attacker compromises the key then there is a chance to lose the security, so to overcome this we proposed Searchable Cryptosystem, using this system user A upload the keywords along with encrypting data and later user B can first search with the related keywords in cloud storage which is upload by user A then if any keywords are match then user B gets encrypted results and performing the decryption with respective key for restricting the attacker to accessing the cloud data.

About Project

Abstract

Cloud storage outsourcing has become a popular application for enterprises and organizations to reduce the burden of maintaining big data in recent years. However, in reality, tenants may not entirely trust the cloud storage servers and may prefer to encrypt their data before uploading them to the cloud server in order to protect the data privacy.

Problem Statement

In Searchable Cryptosystem we are using public key encryption with keyword search which is very useful in many applications of cloud storage. In public key encryption, user’s can encrypt the data ( keywords ) with public key PK and decrypt the data with private key SK.

Proposed System

To overcome inside keyword guessing attack we define a new variant of the smooth projective hash functions (SPHFs). In this system user’s can generate the public key with the help of private key, so even if attacker knows the public key then he can’t accessing the cloud data. Because using of public key attacker can’t generate private key.
